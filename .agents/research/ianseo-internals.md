## 1. Bootstrap & Includes

### Entry point chain

Every page begins with:

```php
require_once(dirname(__FILE__) . '/config.php');   // or a relative variant
```

`config.php` (root) performs:

1. Sets `$CFG->DOCUMENT_PATH` (absolute filesystem), `$CFG->ROOT_DIR` (web-root
   relative), `$CFG->INCLUDE_PATH`, `$CFG->FONT_PATH`.
2. `require_once('Common/distro.inc.php')` – defines `ProgramName`,
   `ProgramVersion`, `ProgramBuild`.
3. `require_once('Common/config.inc.php')` – MySQL credentials
   (`$CFG->DB_HOST`, `$CFG->DB_USER`, `$CFG->DB_PASS`, `$CFG->DB_NAME`), plus
   write-server fields and `$CFG->LANG` (default UI language).
4. `require_once('Common/Fun_DB.inc.php')` – DB abstraction layer.
5. `require_once('Common/Globals.inc.php')` – session management, tournament
   helpers, error display, all `get_text()` / i18n.
6. `require_once('Common/Lib/Fun_Modules.php')` – `module_exists()`,
   `getModuleParameter()`.
7. Sets timezone to UTC, starts `session_start()`, calls
   `define_session_flags()`.

### Special constants on `$CFG`

| Constant             | Value | Meaning             |
| -------------------- | ----- | ------------------- |
| `$CFG->DERANKING`    | 32000 | Archer penalty rank |
| `$CFG->DISQUALIFIED` | 31000 | DSQ code            |
| `$CFG->DIDNOTSTART`  | 30000 | DNS code            |
| `$CFG->DIDNOTFINISH` | 29999 | DNF code            |

## 2. Database Abstraction Layer (`Common/Fun_DB.inc.php`)

### Connections

- **Read server**: `safe_r_con()` → returns a `mysqli` connection used for
  SELECT queries.
- **Write server**: `safe_w_con()` → separate connection for INSERT / UPDATE /
  DELETE.
- Both set `sql_mode = 'NO_UNSIGNED_SUBTRACTION'` and timezone to PHP's
  `date_default_timezone_get()`.
- Charset: both call `set_charset('utf8')`.

### Escaping

```php
StrSafe_DB(string $str, bool $RemoveQuotes = false): string
```

- Escapes with `mysqli_real_escape_string`.
- By default wraps the result **in single quotes** (`'...'`).
- If `$RemoveQuotes = true`, returns the raw escaped string (no surrounding
  quotes).

### Query Execution

| Function                 | Purpose                                   |
| ------------------------ | ----------------------------------------- |
| `safe_r_SQL($query)`     | Read-only query; returns result set       |
| `safe_w_SQL($query)`     | Write query; returns result set           |
| `safe_fetch($rs)`        | Fetch one row as **object** (`$row->Col`) |
| `safe_fetch_assoc($rs)`  | Fetch one row as associative array        |
| `safe_num_rows($rs)`     | Number of rows in result set              |
| `safe_free_result($rs)`  | Free result                               |
| `safe_w_last_id()`       | `mysqli_insert_id` on write connection    |
| `safe_w_affected_rows()` | Affected rows on write connection         |

### Transactions

```php
safe_w_BeginTransaction();
safe_w_Commit();
safe_w_Rollback();
// Also: safe_w_MultiSql($sql) – executes semicolon-separated statements.
```

### Global Key-Value Store

```php
GetParameter(string $name): string          // reads from Parameters table
SetParameter(string $name, string $value)   // writes to Parameters table
```

### Per-Tournament Module Parameters

```php
getModuleParameter($module, $param, $default, $TourId)
// Reads from ModulesParameters table, auto-unserializes if value was serialized.
```

## 3. Session Flags & Tournament Context

### `define_session_flags()` (Globals.inc.php:315)

Called by `config.php` on every page load. Reads the current tournament row
from the `Tournament` table and populates `$_SESSION` with:

| ------------------ | ------------------------ |
| `TourId`           | `ToId`                   |
| `TourName`         | `ToName`                 |
| `TourWhere`        | `ToWhere`                |
| `TourWhenFrom`     | `ToWhenFrom`             |
| `TourWhenTo`       | `ToWhenTo`               |
| `TourType`         | `ToType`                 |
| `TourLocRule`      | `ToLocRule`              |
| `TourSubRule`      | `ToTypeSubRule`          |
| `TourCode`         | `ToCode`                 |
| all ToOptions keys | unserialized `ToOptions` |

### `Set_Tournament_Option($key, $value)` (Globals.inc.php:340)

Reads `ToOptions` from DB, unserializes, sets `$arr[$key] = $value`, re-serializes,
writes back, and sets `$_SESSION[$key] = $value`.

### `CheckTourSession($PrintCrack, $popup)` (Globals.inc.php:364)

Guard function that verifies a tournament is selected:

- Checks `$_SESSION['TourId'] > 0` and that `TourName`, `TourWhere`, dates exist.
- If not valid, either prints an error (non-popup) or outputs JS `window.close()`
  (popup mode).

## 5. Setup Script Anatomy

### 5.1 Lifecycle (`Common/Fun_ScriptsOnNewTour.inc.php`)

`GetSetupFile($TourId, $ToType, $Lang, $SubRule, $subRuleName)` is called when
the user creates or resets a tournament. It:

1. **Deletes all existing tournament-specific data** from:
   `Divisions`, `Classes`, `SubClass`, `TournamentDistances`,
   `DistanceInformation`, `Eliminations`, `Events`, `EventClass`, `Finals`,
   `FinSchedule`, `TeamFinals`, `TargetFaces`.

2. **Includes the setup PHP file** using this resolution order:

   ```
   Modules/Sets/{$Lang}/Setup_{$ToType}_{$Lang}.php   ← preferred (local set)
   Modules/Sets/FITA/Setup_{$ToType}.php               ← fallback (WA default)
   ```

3. **Cleans up orphaned Entries** — resets `EnDivision`, `EnClass`, `EnAgeClass`,
   `EnSubClass`, `EnTargetFace` where referenced records no longer exist.

4. **Stamps rule metadata** — updates `{prefix}TourRules` column on all
   created rows in Divisions, Classes, TournamentDistances,
   DistanceInformation, Events, EventClass, TargetFaces with
   `'{$Lang}|{$typeName}|{$subRuleName}'`.

### 5.2 Setup Script Contract

Variables **available** to the setup script (set by the caller):

| Variable    | Type   | Description                                         |
| ----------- | ------ | --------------------------------------------------- |
| `$TourId`   | int    | Current tournament ID                               |
| `$TourType` | int    | Tournament type ID (1, 3, 6, etc.)                  |
| `$SubRule`  | string | Sub-rule code (e.g. `'1'`, `'Poland-TeamsTop3of4'`) |

Variables the setup script **must set**:

| Variable                  | Type   | Description                                 |
| ------------------------- | ------ | ------------------------------------------- |
| `$tourDetTypeName`        | string | Human-readable name (e.g. `'70m Round'`)    |
| `$tourDetNumDist`         | int    | Number of distances                         |
| `$tourDetNumEnds`         | int    | Number of ends per distance                 |
| `$tourDetMaxDistScore`    | int    | Max score for a single distance             |
| `$tourDetMaxFinIndScore`  | int    | Max individual finals score                 |
| `$tourDetMaxFinTeamScore` | int    | Max team finals score                       |
| `$tourDetCategory`        | int    | Bitmask: 1=Outdoor, 2=Indoor, 4=Field, 8=3D |
| `$tourDetElabTeam`        | int    | Team elaboration? (0/1)                     |
| `$tourDetElimination`     | int    | Elimination rounds? (0/1)                   |
| `$tourDetGolds`           | string | Golds column label                          |
| `$tourDetXNine`           | string | X/9 column label                            |
| `$tourDetGoldsChars`      | string | Golds counted chars (e.g. `'X10'`)          |
| `$tourDetXNineChars`      | string | X/9 counted chars                           |
| `$tourDetDouble`          | int    | Double scoring? (0/1)                       |
| `$DistanceInfoArray`      | array  | Distance information definitions            |

The script then typically calls `require_once('Setup_Target.php')` (either its
own or FITA's) which uses `lib.php` functions to build the tournament structure.

### 5.3 File Naming Convention

For a PL set to handle tournament type 3:

- **`Setup_3_PL.php`** — PL-specific setup for type 3 (70m Round).
- **`Setup_Target.php`** — shared setup for all target types (called from
  `Setup_3_PL.php`).

If `Setup_3_PL.php` does not exist, the engine falls back to
`FITA/Setup_3.php`.

### 5.4 `.set` File Discovery (Legacy/Alternative)

`Tournament/GetLocRules.php` also checks for `.set` marker files:

```
Modules/Sets/{CODE}/Setup_{type}_{CODE}.set
```

These are **empty marker files** that signal a setup is available for the
tournament creation dropdown. Currently no `.set` files exist in the
distribution; `sets.php` is the primary registration mechanism.

### 6.2 Functions

#### `CreateDivision($TourId, $Order, $Id, $Description, $Athlete, $RecDiv, $WaDiv, $IsPara)`

Inserts a row into the `Divisions` table.

| Param          | Type   | Example                             |
| -------------- | ------ | ----------------------------------- |
| `$Id`          | string | `'R'`, `'C'`, `'B'`                 |
| `$Description` | string | `'Recurve'`                         |
| `$Athlete`     | int    | `1` (individual), `0` (not athlete) |
| `$RecDiv`      | string | Records division code               |
| `$WaDiv`       | string | WA division code                    |
| `$IsPara`      | int    | `0` or `1`                          |

#### `CreateClass($TourId, $Order, $From, $To, $Sex, $Id, $ValidClass, $Description, $Athlete, $AlDivision, $RecCl, $WaCl, $IsPara)`

Inserts a row into the `Classes` table.

| Param           | Type   | Example                                            |
| --------------- | ------ | -------------------------------------------------- |
| `$From` / `$To` | string | Birth-year range (e.g. `'2005'`, `'2009'`)         |
| `$Sex`          | int    | `1` = Male, `2` = Female                           |
| `$Id`           | string | `'M'`, `'W'`, `'U21M'`                             |
| `$ValidClass`   | int    | `1` = valid for ranking                            |
| `$AlDivision`   | string | Comma-separated allowed divisions (e.g. `'R,C,B'`) |

#### `CreateSubClass($TourId, $Order, $Id, $Description)`

Inserts a row into the `SubClass` table.

#### `CreateDistanceNew($TourId, $Type, $CategoryFilter, $Distances)`

Creates distance records in `TournamentDistances`.

- `$Type` – tournament type.
- `$CategoryFilter` – a string matched against concatenated `DivId+ClId`.
  Supports `%` wildcard (SQL `LIKE`). Examples: `'_M'` (any div + class M),
  `'RM'` (div R + class M), `'C%'` (div C + any class).
- `$Distances` – array of distance values.

#### `CreateEvent($TourId, ...)` / `CreateEventNew($TourId, $Code, $Description, $Order, $Options)`

Creates an event in the `Events` table with 30+ parameters (match mode,
elimination rounds, target type, team configuration, arrow count, set scoring, etc.).

`CreateEventNew` is the newer API using an associative `$Options` array
with sensible defaults.

Key options:

- `'EvFinalFirstPhase'` – first finals phase (use `FINAL_*` constants).
- `'EvMatchMode'` – `0` = cumulative, `1` = set system.
- `'EvTeamEvent'` – `0` = individual, `1` = team.
- `'EvMatchArrowsNo'` – arrows per end in matches.
- `'EvElimEnds'` – ends in elimination round.
- `'EvElimArrows'` – arrows per end in elimination round.
- `'EvElimSO'` – shoot-off arrows in elimination.
- `'EvFinEnds'` – ends in finals.
- `'EvFinArrows'` – arrows per end in finals.
- `'EvFinSO'` – shoot-off arrows in finals.
- `'EvMixedTeam'` – mixed team event? (0/1).

#### `InsertClassEvent($TourId, $Team, $Number, $Code, $Division, $Class, $SubClass)`

Links an event to a Division+Class combination in the `EventClass` table.
Validates that the Division and Class exist before inserting.

- `$Team` – `0` = individual, `1` = team.
- `$Number` – number of athletes (for team events).
- `$Code` – event code.
- `$Division` / `$Class` / `$SubClass` – must match existing rows.

#### `CreateFinals($TourId)` / `CreateFinalsInd($TourId)` / `CreateFinalsTeam($TourId)`

Generates all finals bracket entries from the `Phases` and `Grids` tables.
Iterates over events whose `EvFinalFirstPhase > 0` and creates `Finals` /
`TeamFinals` rows.

#### `CreateTargetFace($TourId, $Id, $Name, $Classes, $Default, $T1, $W1, ...)`

Creates a target face configuration in `TargetFaces`. Up to 8 target types can
be specified per face:

- `$Classes` – comma-separated class IDs; can use `'REG-'` prefix for regex
  matching.
- `$Default` – `1` if this is the default target face.
- `$T1` / `$W1` – target type ID / weight (1.0 normal).

#### `TargetFaceGoldsXnines($TourId, $Id, $Golds, $Xnine, ...)`

Updates gold/X counting rules for a target face in `TargetFaces`. Specifies
which score characters count as "Golds" and "X/9" for each of the 8 target
slots.

#### `CreateDistanceInformation($TourId, $Distances, $Targets, $Athletes, $Session, $SesName)`

Creates per-distance information rows in `DistanceInformation`. Defines ends,
arrows per end, scoring method per distance. Calls `insertSession()` for
session setup.

Parameters from `$DistanceInfoArray` entries contain:

- Number of ends
- Arrows per end
- Distance values
- Scoring type

#### `UpdateTourDetails($TourId, $Field, $Value)`

Updates a single field in the `Tournament` table.

## 7. Events & Finals Pipeline

### Event Creation Flow

1. Setup script calls `CreateEventNew()` for each event (individual + team).
2. `InsertClassEvent()` links division/class combinations to events.
3. `CreateFinals()` builds bracket structures from the events + `Phases`/`Grids`
   tables.

### Match Mode

- **Cumulative** (`EvMatchMode = 0`): total score wins.
- **Set system** (`EvMatchMode = 1`): set points (WA Olympic-style).

### Finals Phase Constants

See section 6.1 — the `FINAL_*` constants define from which round the finals
bracket starts. For example, `FINAL_FROM_8 = 224` means the bracket starts
from the quarterfinals (1/8).

## 8. Target Face Configuration

Created via `CreateTargetFace()` + `TargetFaceGoldsXnines()`.

Each target face can have up to 8 alternative target types with weights:

```php
CreateTargetFace($TourId, 'TF1', 'Standard 122cm',
    'RM,RW',      // Classes that use this face
    1,            // Default
    TGT_OUT_FULL, 1.0,  // type 1, weight 1
    0, 0,         // type 2 (empty)
    ...
);
```

The `$Classes` parameter supports:

- Exact match: `'RM,RW,CM'`
- Regex match: `'REG-R.'` (matches all classes starting with R)

## 10. Rank Override Mechanism (`Obj_RankFactory`)

### How It Works

`Common/Lib/Obj_RankFactory.php` implements a factory pattern for ranking
classes. When ranking is calculated, it resolves the correct class using this
search order:

```
1) Modules/Sets/{LocRule}/Rank/Obj_Rank_{Family}_{Type}_{SubRule}.php
2) Modules/Sets/{LocRule}/Rank/Obj_Rank_{Family}_{Type}.php
3) Modules/Sets/{LocRule}/Rank/Obj_Rank_{Family}.php
4) Common/Rank/Obj_Rank_{Family}_{Type}_{SubRule}.php
5) Common/Rank/Obj_Rank_{Family}_{Type}.php
6) Common/Rank/Obj_Rank_{Family}.php
```

After finding the main class, it similarly searches for a `_calc` variant:

```
1) ...Obj_Rank_{Family}_{Type}_{SubRule}_calc.php
2) ...Obj_Rank_{Family}_{Type}_calc.php
3) ...Obj_Rank_{Family}_calc.php
4) (same in Common/Rank/)
```

### Family Values

| Family         | Purpose                                      |
| -------------- | -------------------------------------------- |
| `DivClass`     | Qualification by Division/Class (individual) |
| `DivClassTeam` | Qualification by Division/Class (team)       |
| `SubClass`     | Qualification by SubClass                    |
| `Abs`          | Absolute qualification ranking               |
| `AbsTeam`      | Absolute team qualification ranking          |
| `ElimInd`      | Elimination round (individual)               |
| `GridInd`      | Finals bracket (individual)                  |
| `GridTeam`     | Finals bracket (team)                        |
| `Snapshot`     | Snapshot ranking                             |
| `Robin`        | Round-robin ranking                          |

### Existing PL Rank Override

`PL/Rank/Obj_Rank_DivClassTeam_calc.php` extends `Obj_Rank_DivClassTeam` to
implement "best 3 of 4" team scoring:

- Queries `TeamComponent JOIN Qualifications`.
- Takes best 3 archers by `QuScore` / `QuGold` / `QuXnine`.
- Updates `Teams` table with recalculated sums.
- Ranks by `TeScore DESC`, `TeGold DESC`, `TeXnine DESC`.

## 11. HTML Template Pattern

### head.php / tail.php

Pages that render full HTML use:

```php
$PAGE_TITLE = 'My Page Title';
$IncludeJquery = true;   // optional: load jQuery
$IncludeFA = true;       // optional: load Font Awesome
$JS_SCRIPT = [];         // additional CSS/JS includes
require_once('Common/Templates/head.php');

// ... page content ...

require_once('Common/Templates/tail.php');
```

`head.php`:

- Outputs `<!DOCTYPE html>`, `<html>`, `<head>` with standard CSS/JS.
- Appends cache-buster timestamps to CSS/JS files.
- Calls `InfoTournament()` to show the tournament info bar.
- Calls `PrintMenu()` to render the navigation menu.
- Opens `<div id="Content">`.

`tail.php`:

- Closes `</div>`, outputs modal placeholder and `</body></html>`.
- Evaluates `$POST_TAIL` if set.

## 13. Internationalization (i18n)

### `get_text($key, $section)`

Returns a translated string. Translations live in:

```
Common/Languages/{langcode}/{Section}.php
```

Each file returns/populates `$text['key'] = 'Translation';`.

For PL set labels (e.g. `'Setup-PL'`), translations should be added to:

```
Common/Languages/{lang}/Install.php
```

**Note:** The PL module should NOT modify language files outside its directory.
Use `get_text()` keys that fall back gracefully (the key itself is returned if
no translation exists).

## 17. Development Patterns for PL Module

### Adding a New Tournament Type Setup

1. Add the type ID to `$SetType['PL']['types']` in `sets.php`.
2. Create `Setup_{typeId}_PL.php` (e.g. `Setup_3_PL.php`).
3. In the setup script: set all `$tourDet*` variables, then either include
   `FITA/Setup_Target.php` (to reuse standard divisions/classes) or create
   custom divisions/classes via `lib.php` functions.
4. Optionally create a PL-specific `lib.php` for helper functions.

### Adding a Custom Ranking

1. Create `Rank/Obj_Rank_{Family}_calc.php` (or
   `Rank/Obj_Rank_{Family}_{Type}_calc.php` for type-specific overrides).
2. Extend the base class from `Common/Rank/`.
3. Override the `calculate()` method.

### Adding Menu Items

1. In `menu.php`, guard with `$_SESSION['TourLocRule'] == 'PL'`.
2. Add entries to `$ret['{CATEGORY}']`.

### Constraint: Only Modify `Modules/Sets/PL/`

All PL module code **must** live under `Modules/Sets/PL/`. No modifications to
core ianseo files, FITA files, or Common files are permitted.
